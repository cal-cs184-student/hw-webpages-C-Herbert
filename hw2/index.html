Part 4: Edge flip

To implement the edge flip operation, I began by carefully plotting out which elements of the 'before' mesh would correspond to those of the 'after' mesh. While I initially did this with pen and paper, I've included a cleaned up diagram to convey the mapping I used:

With this diagram in hand, and a list of the fields contained in each data type, I traced out what relations would need to be updated. I started with the halfedges, treating 'i' as the given edge and defining every other halfedge in terms of next and twin operations on 'i'. With the halfedges defined, I was then able to pick which halfedges I would assign to the vertices, edges, and faces after applying the operation, completing the mapping.

Once I had double checked everything, implementing the operation as code was actually quite straightforward. Essentially, we just define variables for each element in the mapping, then apply the depicted relationships to the halfedges. Once the halfedges are done, we update the halfedge references in each face, as well as those of the vertices. Note that, with the above mapping, we don't actually have to update the edges, thus, we're done. For this operation, I specifically wanted to avoid the setNeighbors() function, since I felt the above plan was complete enough to produce a cleaner set of individual pointer updates. Thankfully, this turned out to be true, and there was little to no debugging for this part.

Part 5: Edge split

Solving this part followed a process quite similar to that of Part 4.
